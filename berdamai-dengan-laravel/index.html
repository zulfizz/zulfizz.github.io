Berdamai Dengan Laravel

Posted 21. June 2020. 4 min read.

Bermain dengan Laravel? Siapa takut! Ini adalah kali pertama dimana saya berhasil berdamai dengan Laravel setelah sebelumnya gagal dimasa-masa PDKT dengannya. Oiya, laravel adalah salah satu framework atau kerangka kerja PHP terbaik dan populer diseluruh dunia. Bagaimana tidak, selain menggunakan konsep MVC (Model View Controller) yang mudah dipahami dan mudah untuk dipelajari siapapun, Laravel juga dibangun dibawah lisensi MIT yang berarti semua orang bisa menggunakannya dengan bebas.

Kenapa judulnya Berdamai dengan Laravel? Jadi sebenarnya si Laravel ini tidak begitu asing bagi saya dari sejak awal-awal belajar PHP. Namun saat itu saya masih begitu sulit untuk memahaminya karena memang basic di PHP belum terlalu bagus. Sehingga muncul perasaan bimbang dalam diri saya sendiri untuk memilih apakah akan mempelajari Laravel ini atau framework yang lain. Hingga akhirnya saya pun langsung membandingkan dua framework PHP yang populer di Indonesia yaitu Codeigniter dan Laravel.

Ketika itu saya masih kelas 12 SMK dimana saat itu semua siswa diwajibkan untuk membuat Tugas Akhir dengan tema "Sistem Informasi". Setelah berdiskusi dengan kelompok dan mengajukan beberapa judul ke guru pembimbing, kelompok saya pun mendapat persetujuan dari guru pembimbing untuk membuat aplikasi Ujian CBT. Sebagai ketua kelompok, langsung saat itu juga saya mencari informasi tentang Codeigniter dan Laravel. Saya mencoba untuk install keduanya dan membandingkannya untuk mencari tahu bagaimana sih isi dari kedua framework tersebut dan mencari manakah yang cocok untuk saya pelajari.

Jujur saat itu saya masih benar-benar awam untuk mempelajari keduanya, apalagi dengan konsep MVC yang saat itu masih asing di telinga. Setelah sukses menginstall Codeigniter di laptop dan mencoba memahami struktur projectnya, saya melanjutkan untuk menginstall laravel. Impresi pertama ketika melihat struktur project atau folder dari Codeigniter saya langsung tertarik dengannya karena penamaan folder maupun filenya yang mudah dipahami bagi orang awam. Namun hal itu berbanding terbalik ketika saya melihat struktur project Laravel, apalagi Laravel menggunakan composer untuk mengelola projectnya. Pilihan pun jatuh kepada Codeigniter untuk melanjutakn Tugas Akhir saya.

Diatas adalah sedikit cerita bagaimana mengapa dulu saya begitu sulit untuk belajar Laravel dan lebih memilih Codeigniter. Namun disini saya tidak akan membahas Codeigniter, melainkan bercerita awal mula saya bisa memulai project dengan Laravel. Saya juga akan membahas beberapa fitur-fitur yang diberikan Laravel sera kelebihan dan kekurangannya.

Sejak awal diterima kerja sebagai Programmer tepatnya satu tahun yang lalu, saya berusaha bagaiman caranya untuk bisa menambah skill saya dibidang pemrograman. Karena saat itu saya hanya menguasai Codeigniter maka dari itu saya berniat untuk mempelajari Laravel sebagai framework kedua. berbekal pemahaman konsep MVC dari Codeigniter, saya sedikit lebih mudah untuk memahami bagaimana Laravel bekerja. Namun saya masih kesulitan untuk mengimplementasikannnya ke project Laravel secara langsung hingga akhirnya saya memutuskan untuk membeli sebuah Ebook Laravel 6. Oiya, sebelumnya juga sudah membaca beberapa artikel yang membahas Laravel dan menonton tutorialnya di Youtube.

Dengan membeli ebook pun tidak membuat saya langsung bisa menguasai Laravel. Butuh berminggu-minggu untuk memahaminya karena memang isinya berbeda jauh dengan Codeigniter, apalagi pada bagian relationship yang cukup membuat saya bingung. Setelah menyeleseikan project yang ada di ebook tersebut saya berhenti menggunakan Laravel sampai beberapa bulan. Alasannya karena saya merasa sulit sekali untuk memahami Laravel ini meskipun beberapa tutorial sudah saya coba namun hasilnya belum maksimal.

Setelah beberapa bulan meninggalkan Laravel, muncul dorongan dari dalam diri saya untuk mempelajari kembali framework ini. Saya mengubah pola pikir bahwa saya bisa dan yakin untuk mempelajari framework ini yang sebelumnya saya beranggapan Laravel sulit untuk dipelajari. Saya langsung membuat project Laravel baru di laptop dan mempelajarinya dari awal lagi dengan semangat yang berlipat-lipat dari sebelumnya :D. Dimulai dari instalasi project, koneksi ke database, menggunakan artisan, membuat migration dan lain lain. Anehnya saya tidak merasa kesulitan ketika melakukannya satu per satu dan langsung bisa memahaminya. Mungkin karena pandangan saya terhadap framework ini sudah berubah.

Karena tidak ada ide untuk membuat project, akhirnya saya memutuskan untuk mengubah project Tugas Akhir saya dulu yang awalnya menggunakan Codeigniter menjadi Laravel dengan frontendnya menggunakan Vue, tentunya ada beberapa perubahan dan perbaikan dari segi fitur dan menu. Lagi-lagi saya tidak merasa kesulitan dengannya. Saya merasa nyaman ketika menggunakan fitur migration, authentication, dan fitur-fitur lainnya.

Tak lama setelah itu saya mendapat project baru di kantor untuk membuat sebuah CMS menggunakan Codeigniter karena basic saya di kantor adalah programmer Codeigniter. Namun saya menawarkan diri untuk menggunakan Laravel sebagai frameworknya yang akhrinya hal tersebut disetujui oleh project manager saya. Banyak hal baru yang bisa saya pelajari ketika mengguanakan Laravel.

Setelah saya bercerita panjang lebar tentang Laravel, sebenarnya apa saja si fitur-fitur yang diberikan Laravel kepada penggunanya?

  1. Blade Template Engine
    Laravel menggunakan Blade sebagai template enginenya. Hal ini tentu memudahkan programmer dan bisa mempercepat proses pengembangan aplikasi.
  2. Eloquent ORM
    Fitur ini berfungsi untuk menjalankan operasi database pada aplikasi dan sudah mendukung beberapa database seperti MyQSL, PostgreSQL, dll.
  3. Query Builder
    Sama seperti Eloquent ORM, Query Builder manawarkan fitur untuk mengelolan maupun menjalankan operasi pada database dengan mudah.
  4. Authentication
    Untuk membuat sistem autentikasi di Laravel sangat mudah dan hanya perlu menjalankan beberapa perintah saja. Selain itu juga mudah untuk disesuaikan dengan kebutuhan sistem.
  5. Routing
    Semua request pada Laravel dipetakan dalam sebuah route. Kita bisa mengelompokan route, menamainya, dan menambahkan filter didalamnya.
  6. Modularity
    Laravel menggunakan modul-modul yang dapat diatur dengan menggunakan composer sehingga memudahkan dalama mengelolanya.
  7. Testability
    Framework ini hadir dengan menggunakan PHPUnit yang berfungsi untuk menguji aplikasi yang kita kembangkan.
  8. Caching
    Laravel sudah support Redis untuk sistem cachingnya. Redis adalah aplikasi yang menyimpan key value dengan tipe strings, hashes, lists, sets, dan sorted sets.
  9. Dan Lain-lain

Masih banyak fitur-fitur yang Laravel hadirkan, tentunya hal ini sangat memenjakan programmer. Dengan kemudahan-kemudahan tersebut Laravel mampu menjadi framework PHP populer di dunia.
